# liv_ay 플랫폼 개발비용 추정 보고서

> 4개 저장소(liv_ay_backend / liv_ay_frontend / liv_ay_manager / chofam)를 전통적인 팀이 처음부터 새로 개발했을 경우의 인월(MM) 및 비용 추정치. 현재 main 브랜치 코드베이스 규모와 도메인 복잡도를 근거로 산정했다.

- **기준일**: 2026-07-15
- **대상**: liv_ay_backend · liv_ay_frontend · liv_ay_manager · chofam (모두 main 기준)
- **산정 방식**: 기능 단위 상향식 인월(MM) 산정 + 코드 규모 교차 검증
- **통화**: KRW

## 총괄 추정액

노임 단가 가정에 따라 세 구간으로 제시한다. 개발 직접 공수(47.5 MM)에 기획·디자인·PM(20%), QA·통합(10%)을 더한 총 61.8 MM을 기준으로 계산했다.

| 구간 | 금액 | 산정 기준 |
|---|---:|---|
| 낮음 | 약 4.4억 원 | 중급 단가 기준 (임금 -20%) |
| **중간 (채택)** | **약 5.5억 원** | 고급/중급 혼합 단가 · 61.8 MM |
| 높음 | 약 6.9억 원 | 특급 단가·외주 프리미엄 (임금 +25%) |

> **산정 성격에 대한 안내.** 본 수치는 공개된 소프트웨어 노임단가 통념과 코드 규모·도메인 난이도에 대한 판단을 결합한 근사 추정치다. 실제 계약 견적은 팀 구성, 지역, 계약 형태(SI 외주/내재화), 최신 노임단가 고시에 따라 달라진다.

## 산정 방법론

표준 LOC/인월 비율 대신, 각 저장소의 기능 도메인을 나눠 상향식(bottom-up)으로 공수를 산정한 뒤 코드 규모로 교차 검증했다. 노임 단가는 SW기술자 등급별 통상 단가 통념을 참고해 3단계로 가정했다.

| 등급 | 적용 대상 | 월 단가(가정) |
|---|---|---:|
| 중급 | 표준 CRUD·관리자 도구 개발 (manager, chofam) | 7,000,000원 |
| 고급 | AI 파이프라인·결제정산·게임 엔진 개발 (backend, frontend) | 9,000,000원 |
| 특급 | 기획·아키텍처·PM | 11,000,000원 |

## 가정 및 산정에서 제외한 항목

- 노임 단가는 2026년 SW기술자 평균임금 통념을 참고한 근사치이며, 최신 고시 단가와 다를 수 있다.
- 인프라 운영비(GCP/Firebase/Supabase 클라우드 비용), API 사용료(Vertex AI, 이미지/영상 생성 모델), 앱스토어 심사·마케팅 비용은 포함하지 않았다.
- Flutter 자동 생성 다국어 파일(`app_localizations.dart`, 7,321줄)은 라인 수 집계에서 제외했다.
- 게임 스토리·캐릭터 설정 등 콘텐츠 저작(작가 비용)은 소프트웨어 개발비 범위 밖으로 간주해 제외했다.
- 유지보수·운영 인력 비용은 별도이며, 본 추정은 최초 구축(1회성 개발) 비용만 다룬다.
